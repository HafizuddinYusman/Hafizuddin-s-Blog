Database Design

Log #1: Designing Relational Schemas and Enforcing Data Integrity

Published on: April 14, 2026

A critical phase in system engineering is building database tables that resist anomalies. During my recent architectural work, I focused heavily on database design, normalizing schemas to Third Normal Form (3NF) and drafting strict entity relationships.

When migrating schemas or writing custom SQL column mappings, attention to naming accuracy is vital. For example, troubleshooting mismatched column markers like tracking accurate time-stamps (e.g., handling explicit fields like Appointments_DateTime) taught me the importance of maintaining rigorous data dictionaries to eliminate unexpected framework execution blocks.

System Analysis

Log #2: Structural Analysis Using Multi-Level Data Flow Modeling

Published on: May 02, 2026

Software systems fail when boundaries are poorly defined. This post explores system analysis methodologies, contrasting rigid Waterfall steps with fast Agile sprints. I spent considerable time defining project scope through visual representation.

A major takeaway was understanding diagram scope: correctly identifying high-level boundaries as Context Diagrams before attempting to break down deeper structural flows into Level-0 and Level-1 Data Flow Diagrams (DFDs). Getting these models right makes documenting functional system specs significantly more straightforward.

Backend Engineering

Log #3: MVC Architecture and Routing in Web Frameworks

Published on: May 25, 2026

Implementing real-time dashboards and scheduling views requires solid framework configuration. This log maps out how I handled routing structures, models, and custom controllers inside an MVC (Model-View-Controller) environment like PHP's CodeIgniter framework.

I detailed methods for routing clean URLs directly to patient dashboards, handling incoming data arrays, and intercepting missing route errors. Structuring applications using clean MVC divisions isolates rendering layouts from core backend logic—making the entire application easier to test and scale.
